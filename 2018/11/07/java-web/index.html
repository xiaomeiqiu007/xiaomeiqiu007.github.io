1.SERVERLET

1.1 SERVERLET简介:

  • Serverlet是位于Web服务器内部的，服务端的，独立于平台和协议的java应用程序，可以生成动态的web。
  • Servlet是一个可信赖的程序，不受java安全性的控制，拥有和普通java应用程序一样的权限。
  • Servlet是CGI Script的一种替代技术，由于Servlet在性能、可移植性、代码重用等方面比CGI具有显著的优势，因此在未来的技术发展过程中，Servlet有可能彻底取代CGI。

1.2 Servlet优点：

  • Servlet 在 Web 服务器的地址空间内执行。这样它就没有必要再创建一个单独的进程来处理每个客户端请求。
  • Servlet 是独立于平台的，因为它们是用 Java 编写的
  • 服务器上的 Java 安全管理器执行了一系列限制，以保护服务器计算机上的资源。因此，Servlet 是可信的。
  • Java 类库的全部功能对 Servlet 来说都是可用的。它可以通过 sockets 和 RMI 机制与 applets、数据库或其他软件进行交互。

1.3 Serverlet运行原理：

（1）客户向服务器发送对页面的请求。

（2）如果Servlet还没有装入，服务器就装入它。

（3）服务器把请求信息送给Servlet，给每一个请求创建一个执行的新线程（Java语言的线程允许同时执行多个任务。

（4）Serlvet处理这个请求，生成一个响应并传递给服务器。

（5）服务器把响应送回给客户。

1.4 Serverlet请求处理

1、getMethod();获取请求提交的方法名称；
2、getRequestURI();  获取请求的详细资源；
3、getQueryString(); 获取请求提交的数据；
4、getProtocl();获取协议的协议名及版本号；
5、getContextPath();获取请求 URI资源的上下文路径；
6、getServletPath();获取Servlet的映射路径；
7、getRemoteAddr();来访者的IP地址；
8、getLocalAddr();返回WEB服务器的IP地址；
9, getHeader(String key):通过key获取指定的请求头value
10 getHeaders(String name) :通过key获取指定的请求头
11 getHeaderNames() :获取所有的请求头的名称

1.5 serverlet转发与重定向

  • forward：转发
  • redirect：重定向
  • include：包含

转发： 将当前request和response对象保存，交给指定的url处理。并没有表示页面的跳转，所以地址栏的地址不会发生改变。（只能在服务器内部进行转发） 工作原理：

  • 客户首先发送一个请求到服务器端，服务器端发现匹配的servlet，并指定它去执行。
  • 当这个servlet执行完之后，它要调用 getRequestDispacther()方法，把请求转发给指定的URL。
  • 整个流程都是在服务器端完成的，而且是在同一个请求里面完成 的。
  • l整个过程是一个请求，一个响应。

重定向： 包含两次浏览器请求，浏览器根据url请求一个新的页面，所有的业务处理都转到下一个页面，地址栏的地址会变发生改变。 语法： response.sendRedirect(“URL”); 工作原理：

  • 客户发送一个请求到服务器，服务器匹配servlet，这都和请求转发一样。
  • servlet处理完之后调用了sendRedirect()这个方法，这 个方法是response的方法，所以，当这个servlet处理完之后，看到response.senRedirect()方法，立即向客户端返回这个响应，响应行告诉客户端你必须要再发送一个请求，去访问目标URL。
  • 紧接着客户端得到响应后，立刻发出一个新的请求，去请求test.jsp, 这里两个请求互不干扰。
  • 在前面request里面setAttribute()的任何东西，在后面的request里面都获得不了。可见，在 sendRedirect()里面是两个请求，两个响应。

[COOKIE]

COOKIE介绍：
  1. cookie是存储在客户端浏览器目录的文本文件，数据量受限制。
  2. 当浏览器运行时，cookie存储在RAM中。即使客户端和web服务器失去连接，cookie也可以存储在客户端计算机的硬盘中。当客户端结束其浏览器对话时，即终止所有的Cookie。
  3. Cookie是利用网页代码中的HTTP头信息进行传递的。如果浏览器禁用Cookie，则关闭浏览器，Cookie就会消失。
  4. Cookie的保存时间是由其Expires（有效期）属性决定的。服务器可以通过修改Expires属性来修改Cookie的保存时间。如果不设置该属性，则Cookie只在浏览器浏览网页期间有效，一旦关闭浏览器，Cookie会自动消失。
COOKIE的创建与使用：

创建cookie

Cookie c1=new Cookie(“name”,”value”);

向客户端发送创建cookie请求

response.addCookie(c1);

服务器接收请求

Cookie[] cookies=request.getCookies();


void setMaxAge(int expiry)    //设置Cookie的过期时间，单位为称。设置为负值，关闭浏览器，Cookie失效。设置为零，销毁Cookie
void setPath(String path)    //设置Cookie适用的路径
void setValue(String newValue)    //设置Cookie的新值

如果要把Cookie保存在磁盘上，需要使用setMaxAge方法设置Cookie的生存时间，此种Cookie称为持久化Cookie

[SESSION]:

SESSION简介： 1.在JSP中可以直接使用内置的session对象来进行会话跟踪和管理，在Servlet中使用HttpSession接口来实现会话跟踪。 2.服务器通过为客户端创建一个会话对象保存该客户端的信息。当客户端第一次访问服务器时，服务器会分配给客户端一个会话对象以及一个单独的会话ID。在接下来的请求中，每当客户端提交请求时，都会提交本身的ID。服务器通过会话ID判断哪些连接是来自于同一个客户端的连接。 SESSION原理： 1．当客户端向服务器端初次提交请求时，请求中并不包含sessionID。 2．服务器端的Servlet容器会根据getSession( )方法创建一个HttpSession对象，并分配一个sessionID。HttpSession对象保存在服务器端，此时的HttpSession处于新会话状态。 3．服务器将处理请求的响应结果连同sessionID回传到客户端。 4．当客户端再次提交请求时，会将请求连同其sessionID一起提交给服务器。 5．服务器接收到请求以及sessionID后，将请求和保存在服务器端的session对象联系起来，根据sessionID判断哪些连接是来自于同一个客户端的会话范围内的连接。此时的session对象已不是新的会话。 6．循环3-5，直到session失效。 SESSION创建： 获取SESSION，没有则创建一个Session

HttpSession mysession = request.getSession();

SESSION常见方法：

void invalidate( )：使当前关联的session失效，Servlet容器会释放HttpSession对象所占用的资源；
void setAttribute(String name,Object value)：将一对name/value属性值保存在HttpSession对象中；
Object getAttribute(String name)：根据name参数返回保存在HttpSession对象中的属性值；
void removeAttribute(String name)：根据name参数在HttpSession对象中移除属性；
boolean isNew( )：判断当前关联的HttpSession是否是新建对象；
void setMaxInactiveInterval(int interval)：以秒为单位，设置session的有效时间。如果参数为负值，表示session永不失效。
int getMaxInactiveInterval( )：返回session的有效时间。

[jdbc-web]:

由于项目中不少环境之下都是多次使用sql数据库连接，以及查询语句，避免重复操作造成的资源浪费。程序员会项目分为多个类，如果按照逻辑想的话，可以对项目进行分层，我们对数据库操作的可以分为三个层， 数据访问层：通常命名为DAO，通常在这个包中写增删改查。 持久化层：hibernate。 模式层：通常用javabean，利用类和数据表相对应。 数据库连接层：进行数据库连接驱动的加载。通常以config等命名。 com.util.DBConnection.java:主要进行数据库连接，通过getConnection方法，返回连接Connection coon对象，这里我返回的statement对象

package com.util;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.Statement;

//主要目的加载驱动，创建连接池
public class DBConnection {
    private static final String DRIVER="com.mysql.jdbc.Driver";
    private static final String URI = "jdbc:mysql://localhost:3306/security";
    private static final String USER_NAME ="root";
    private static final String USER_PASS = "root";
    private static Connection coon;
    private static Statement state;
    static {
        try {
            Class.forName(DRIVER);
        }catch (Exception e) {
            // TODO: handle exception
            System.out.println("数据库加载驱动一张");
            e.printStackTrace();
        }
    }
    public static Statement getConnection() {
        try {
             coon = DriverManager.getConnection(URI, USER_NAME, USER_PASS);
        }catch (Exception e) {
            // TODO: handle exception
            e.printStackTrace();
            System.out.println("数据库连接失败");
        }
        try {
            state= coon.createStatement();
        }catch (Exception e) {
            // TODO: handle exception
            System.out.println("statement创建失败");
            e.printStackTrace();
        }
        return state;
    }

}

package com.Dao;

public class UserDao {
